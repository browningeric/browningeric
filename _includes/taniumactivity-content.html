Tanium Activity

Conveying real-time visibility in a manageable way

How can we show real time data for huge on-prem and cloud networks from anywhere?

Tanium excelled at capturing and working with real-time data, but the interface to use the platform was a menagerie of largely disparate and static views relying on domain knowledge for where to go and what to look for.

I was given the task of designing a solution to provide insight into what is currently happening in the customer's environment.

Responsibilities
  • user and stakeholder research
  • UI design
  • prototyping
  • executive presentations

What's important to users?

I interviewed a variety of users to understand what they needed and why our current solutions weren't working.

Interviewees

  • Current users (IT Administrators)

Further detail

Digging deeper into the data, a few additional themes emerged.

Users needed aggregate data as many environments manage > 100,000 endpoints; displaying individual endpoint data would be way too noisy to be useful, and that's available elsewhere in the platform.

Rather than all data (<~100 types of activity), users were interested in a handful of highly impactful tasks like machines performing scans, software updates being installed, etc.

Prioritization and Design

We decided to focus on two broad areas of data: deployments (things being pushed to/made to happen on machines), and scans (a requisite step for visibility and compliance with these machines).

Deployments have a number of possible statuses, represented by colors in the bars (with tooltip labels as well). These match existing module functionality, making it easy for users to identify what they are seeing.

For scans the important piece of data is more binary; simply knowing if a machine was performing a given scan or not. With that in mind I used a single color to represent scan activity, which also helps differentiate that tab from deployments.

Progressive Detail

One theme I pushed with this design was offering levels of detail for users to access:

  • A persistent small icon representing simply which endpoints in an environment are doing "something". If a user performs an action they think should result in heavy activity but they don't see that reflected, they know they need to dig deeper.
  • A side panel that has aggregate data at the top of the panel. This allows for a quick judgment for a user to say "this looks okay to me" or "I need to learn more".
  • More context further down in the panels. Breaking apart the data by product module or activity allows for a more specific understanding of what's happening.
  • New page views for the most detail. If a user needs to really dig in to the data, I added links to very detailed page views which mirror the structure of the panel but with a greater level of detail.

Outcome

The activity panel and pages were implemented in the platform, and later rolled into the development of Tanium Feed which built upon this functionality with additional features around curation and recommendations.
